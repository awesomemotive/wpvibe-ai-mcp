# Contributing to WPVibe

Thanks for your interest in contributing to WPVibe *(listed on WordPress.org as "Vibe AI")*, the open-source WordPress plugin for AI assistant integration via the Model Context Protocol.

## Bug reports

If you've found a bug, [open an issue](https://github.com/awesomemotive/wpvibe-ai-mcp/issues/new?template=bug_report.md). Please include:

- WordPress version
- PHP version
- Vibe AI plugin version
- AI client and version (Claude Desktop, Cursor, etc.)
- Hosting environment (local, shared, managed WP, VPS)
- Steps to reproduce
- Expected vs. actual behavior

## Feature requests

[Open a feature request](https://github.com/awesomemotive/wpvibe-ai-mcp/issues/new?template=feature_request.md). Tell us:

- The use case (what are you trying to accomplish?)
- What you've tried so far
- Why existing tools don't solve it

## Pull requests

PRs are welcome. Before submitting:

1. **Open an issue first** if your change is non-trivial — saves both of us time if the change isn't a fit for the project
2. **Match existing code style** — WordPress Coding Standards for PHP
3. **`php -l` clean** on every modified PHP file — CI will check
4. **Update `readme.txt` changelog** for any user-visible change
5. **One concern per PR** — easier to review, easier to revert if needed

## Security vulnerabilities

**Please do not open public issues for security vulnerabilities.** Email **security@wpvibe.ai** with details. We'll respond within 48 hours, work with you on a coordinated disclosure timeline, and credit you in the changelog when the fix ships.

## Code of conduct

Be respectful. We follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
